XHTML vs HTML

  • Case sensitive
  • Void entities: img, br, hr
  • Unexpected end tags: </br> </p>
  • Unescaped & and <
  • CDATA elements: script and style

Some differences

XHTML

<p>Hello</p>
<p>world</p>
<img src="hello?foo=1&amp;bar=2"/>
<script type="text/javascript">
<![CDATA[
... unescaped script content ...
]]>
</script>

HTML

<p>Hello
<p>world
<IMG src="hello?foo=1&bar=2">
<script>
... unescaped script content ...
</script>

XHTML is a bad idea

  • Internet Explorer doesn't support XHTML
  • No parsing error recovery
  • Any JavaScript code must be inside a CDATA block
  • Any entities (&amp;) must be escaped
  • In general, there is nothing to gain by using XHTML
