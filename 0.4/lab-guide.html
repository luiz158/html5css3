How to use this lab guide

The files required to implement these labs can be downloaded as a zip file from GitHub.

Alternatively, they can also be obtained by cloning the repository using Git.

Installing Git

Git can be downloaded for Windows and Mac from the Git website.

There is also a package for Ubuntu:

sudo apt-get install git git-gui gitk

Cloning the repository

Once Git is installed, run git clone:

git clone git://github.com/instructormatters/html5css3.git

This should create the html5css3/labs folder.

Structure

Each lab is divided into two folders, Before and After. Before provides with all files needed to start the exercise. After contains the finished version of the exercise to help you along.

Applications

The applications required for the labs can be found in the Appendix section at the end of this document.

Description

Download the labs folder as described in the Introduction.

Install the required tools for the labs. The installation and configuration instructions for each application can be found at the Appendix. Use them and install:

  • Sublime Text Editor with LESS or SASS package (the instructor will indicate which one should be executed).
  • Node.js and if LESS is selected by the instructor.
  • Ruby if SASS is selected by the instructor.

Now, open Sublime Text Editor and select Project → Open Project. Open labs/labs.sublime-project and execute the following steps to get familiarized with the editor.

Manage files

  • Toggle sidebar on/off: Ctrl + K, Ctrl + B.
  • Open file: Ctrl+P
  • See several files at once: Shift + Alt + [1234589] (choose one number). Files can get aligned side by side, top vs bottom, etc. Once done, you can switch back with Shift+Alt+1.

Simple actions

  • Write any text, select a word, then press  Shift+Alt+W and type b.
  • Prepare a list of data, select all the text and hit Tab to indent it a bit. Now while holding the  Shift key, right click and drag  to select a rectangular selection of code. Type the start tag to get it inserted at the beginning of each line.

Multiple cursors

The previous step is just a concrete example of using multiple cursors:

  • Ctrl + click: create new cursor.
  • Ctrl + double click: create a new cursor selecting the word that was double-clicked.
  • Shift + click: select using the latest cursor (so: ctrl + click, then shift + click to add a selection to the set of cursors).
  • Any keyboard action: Operate on all cursors at once. Keyboard operations (like Ctrl+V) will work, and so will normal typing (like hitting End to write the ending tag at the end of each line).

To test this:

  • Write any text, ctrl+click at the beginning of each line. Write <p> or <li>.
  • Ctrl + double click on several words,then type " (double quote)
  • Ctrl + double click on several words, hit  Shift + Alt + W, then type b

The before folder contains a configured Compass project. To compile, execute in labs/sass/before:

After this, any change in style.scss will trigger a re-compilation automatically.

The CSS stylesheet requires the following changes to remove duplicity:

  • Repeated values can be stored in variables. Extract and use variables for colors (backgrounds, shadows, text color) and the width of links.
  • Replace then the values in the stylesheet with variables. This allows also to show relations between magnitudes, such as the line height of a link and its width. These can also be used when calculating the size of the .links container.
  • Use nested syntax to group the rules of .links, .right and .left in three blocks.
  • Include compass in the stylesheet using @import "compass".
  • Use the replace-text() mixin to perform the image replacements.
  • Use the border-radius() and box-shadow() mixins to avoid writing the vendor prefixes.

If in time, use the color functions complement() or darken() to transform the color of the links on the left into the right.

Modernizr: Polyfill with Modernizr

« Back to index

Duration

40 minutes

Modernizr Customization

The full Modernizr build is too big for real projects. For this lab the student should create a custom Modernizr build including:

  • Input Types
  • Input Attributes
  • hmtl5shiv v3.6
  • Modernizr.load (yepnope.js)

Once downloaded, it should be included using a script tag at the end of the <body> tag. After this tag you can start using yepnope to load the required polyfills.

<script src="modernizr.custom.11198.js"></script> 
<script> 
  yepnope(...); 
</script>

Description

Make the previous lab work in Firefox 3.6 or IE8 using polyfills: all the polyfills needed are in the polyfills folder. You can find more in Cross browser polyfills.

Some of the polyfills require the javascript library jQuery to work. It must be loaded before any polyfills. You can download it from its website, or link it from a 3rd party repository, such as Google Hosted Libraries.

The following features should be implemented:

  • autofocus: use autofocus.js, which does not need initialization.
  • placeholder: Use the jQuery placeholder plugin (by Mathias Bynens) and initialize it indicating the inputs it should process:
    { 
  test: Modernizr.input.placeholder, 
  nope: 'jquery.placeholder.min.js',
  callback: function() {
    $('input').placeholder();
  }
}
  • required and input types: If the browser does not include a native support for required inputs and several input types, as range type, we will use the jQuery plugin (by Andrew Dodson), loading the javascript and its css styles:
    { 
  test: Modernizr.input.required, 
  nope: ['jquery.form.css','jquery.form.js'], 
  callback : function(url, test, key) {
    url == 'jquery.form.js' && $('#form1').form();
  }
}

AppCache

« Back to index

Important

This lab tests the offline behavior of browsers. Use the local server described in the Appendix to test this lab. This lab will not work unless executed using localhost:8000 instead of file:///

Duration

20 min

Description

A simple web application composed by two pages (awesomizer.html and switcher.html) is provided. Create a cache manifest to enable this app to work offline.

In Chrome, it is possible to check the status of the application cache in chrome://appcache-internals. This is very useful to simulate a first request to the local server, or examine the files that have been cached.

The manifest should include all the static files in the CACHE section: the stylesheets and the application scripts. Some of the stylesheets are located in the labs/lib folder. Specify the absolute path for these resources.

In addition to this, the page includes a Twitter widget that consists of widget.js and an external script hosted in widgets.twimg.com. To allow the application to download an external resource, you should add a wildcard (*) in the NETWORK section.

When offline, the script sorry.js should be used instead of widget.js to notice the user that Twitter is not available. Specify this in the FALLBACK section.

Be aware while testing the offline behavior that even if the local server is not running your computer will probably be still connected to the Internet,

Also, remember that anything that is not listed in the manifest (whether in the explicit cached section or not) will not work offline.

-ms-filter:"progid:DXImageTransform.Microsoft.gradient(startColorStr='#9c9c9c',EndColorStr='#7c7c7c')";

LiveReload

LiveReload is a Ruby plugin for Guard that watches a folder and refreshes any modified files in a client browser. If the file is a stylesheet, the browser redraws the HTML document without reloading it. Though not required by any lab, developers can experience a productivity boost by installing it.

  • Windows and Mac: either follow the instructions for Linux or install the LiveReload GUI from livereload.com.

Some labs require a local HTTP server to work. There are two different alternatives to do this.

Node

After configuring Node, you can start a simple HTTP server by executing this in the labs folder:

node server.js

After this, you can open your browser at localhost:8000.

Python

If you have Python installed (it comes included in Linux and Mac by default), you can start a server by executing this in the labs folder:

./server.sh

(You should use server.bat in Windows). After this you can open your browser at localhost:8000.

Windows users may download Python from Python's website. In order to be able to execute Python from the command line, you may to add C:\Python27 to the Path variable (My Computer > Properties > Advanced > Environment Variables) and restart your computer.
