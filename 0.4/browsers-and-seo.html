IE render modes

Quirks mode: render as IE5

Standards mode (Intranet): IE7

Standards mode (Extranet): IE8 or 9

X-UA-Compatible

<!-- render as ie9 -->
<meta http-equiv="X-UA-Compatible" content="IE=9">

<!-- render as ie7 (quirks/standard mode, according to doctype) -->
<meta http-equiv="X-UA-Compatible" content="IE=EmulateIE7">

<!-- render as ie9 or ie7 (avoid ie8) -->
<meta http-equiv="X-UA-Compatible" content="IE=7, IE=9">

X-UA-Compatible values

  • EmulateIE7, EmulateIE8, EmulateIE9: use standards or quirks mode, according to doctype
  • IE7, IE8, IE9: Always use standards mode.
  • IE5: Always use quirks mode
  • edge: Use the best engine available
  • You should avoid using X-UA-Compatible if possible (rely on standards instead)

Chrome Frame

If installed, render using Chrome Frame instead of IE.

META:
<!-- Use Chrome Frame -->
<meta http-equiv="X-UA-Compatible" content="chrome=1">

<!-- Use IE9 (if installed) or Chrome Frame -->
<meta http-equiv="X-UA-Compatible" content="IE=9,chrome=1">
HTTP header:
X-UA-Compatible: chrome=1

Multiple values

Use IE 9 if available, Chrome Frame otherwise:

<meta http-equiv="X-UA-Compatible" content="IE=9,chrome=1">

Install from the server side

Detect if the user has Chrome Frame installed by inspecting the User-Agent:

Mozilla/4.0 (compatible; MSIE 6.0; Windows NT 5.1; SV1; chromeframe/11.0.660.0)
Server side:
if (userAgent.contains("MSIE") && !userAgent.contains("chromeframe")) {
  return redirect("/chromeFrameRequired.html");
}

Pie as JavaScript

  • Blocks the page while loading.
  • Not automatically attached to new elements added dinamically.

  <script type="text/javascript" src="PIE.js"></script>
<![endif]-->
<script>
  if (window.PIE) {
      $('.rounded').each(function() {  PIE.attach(this);  });
  }
</script>

JavaScript vs behavior

  • PROS:
    • You can load PIE.js from a different domain.
    • It is easily integrated within JavaScript libraries.
  • CONS:
    • JavaScript file blocks the page while loading.
    • It is not automatically attached to new elements added dinamically.

Search engines

Search websites according to:

  • Robots.txt
  • Sitemap.xml

Robots.txt

First landing page for a web crawler

Sitemap: http://myhost.com/sitemap.xml
User-agent: *
Disallow: /secure/
Disallow: /tmp/

Security considerations

robots can ignore your /robots.txt, especially malware robots.

Anyone can see which sections of your server are secured.

Sitemap.xml

Inform search engines about pages available for crawling

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
   <url>
      <loc>http://www.example.com/index.html</loc>
      <lastmod>2005-01-01</lastmod>
      <changefreq>daily</changefreq>
      <priority>0.8</priority>
   </url>
</urlset>

Contents missing in the sitemap

Will be indexed as well if they get linked somewhere

So why use sitemaps?

  • Specify frequency of change and priorities
  • Point to otherwise unreachable contents.

PageRank

Algorithm by Larry Page (Google) used to measure the weight of a page in the Internet

Bits included in PageRank

  • Page URL and host name
  • Title and description of the page
  • Website performance
  • Language deduced by the search engine
  • Contents structure
  • Referrals

Bits NOT included in PageRank

  • <meta name=keywords>
  • Language set in the page
  • Invisible contents
  • <a rel="nofollow">

Clean URLs

Use hyphens to separate words, not underscores

/posts/why_seo_rocks WRONG

/posts/why-seo-rocks CORRECT

Meta robots

  • nofollow: do not follow any link on this page.
  • noindex: do not store this page in the search engine index.
  • noarchive: do not store a cached version of the page.
  • There is also follow, index, but they are the default behavior.
